## 二、功能需求對照（README「Features to Implement」）

### ① 建立通知 — `POST /notifications`
| 項目 | 內容 |
|---|---|
| 需求 | 存進 MySQL → 推送 RocketMQ `notification-topic` → 加入 Redis 最近 10 筆快取 |
| 實作 | `NotificationController.create` → `NotificationService.create`（insert → putById → pushRecent → MQ send） |
| 狀態碼 | 201 Created（含 id、version） |
| 測試 | `NotificationControllerTest.create_returns201`、`NotificationServiceTest.create_persists_publishes_caches` |
| 實機 | ✅ 201；MySQL 有列、Redis 有 `notification:{id}`、RocketMQ topic offset 增加 |
| 結果 | ✅ 通過 |

### ② 依 ID 查詢 — `GET /notifications/{id}`
| 項目 | 內容 |
|---|---|
| 需求 | 先讀 Redis，未命中讀 MySQL 並回寫 Redis |
| 實作 | `NotificationService.getById`（cache-aside：命中即回；未命中 → DB → 回寫） |
| 狀態碼 | 200 / 404 |
| 測試 | `NotificationServiceTest.getById_cacheMiss_backfills`、`getById_missing_throws404`、`NotificationControllerTest.getById_*` |
| 實機 | ✅ 200（存在）/ 404（不存在） |
| 結果 | ✅ 通過 |

### ③ 查最近通知 — `GET /notifications/recent`
| 項目 | 內容 |
|---|---|
| 需求 | 回傳最近通知（最多 10 筆） |
| 實作 | `NotificationService.recent`（Redis 優先，未命中由 `findRecent(10)` 重建） |
| 狀態碼 | 200（陣列，含 id/type/recipient/subject/createdAt…） |
| 測試 | `NotificationControllerTest.recent_returns200List`、`NotificationServiceTest.recent_cacheMiss_rebuildsFromDb` |
| 實機 | ✅ 200；30 筆並發建立後清單仍維持 **10 筆**（原子 LTRIM） |
| 結果 | ✅ 通過 |

### ④ 更新通知 — `PUT /notifications/{id}`
| 項目 | 內容 |
|---|---|
| 需求 | 更新 MySQL 的 subject/content；Redis 同步更新或失效；不存在回 404 |
| 實作 | `NotificationService.update`（更新 DB → 刷新 by-id 快取 → recent 失效；不存在 404） |
| 狀態碼 | 200 / 404 /（樂觀鎖衝突 409） |
| 測試 | `NotificationServiceTest.update_refreshesCacheAndInvalidatesRecent`、`update_missing_throws404`、`update_versionConflict_throwsConflict` |
| 實機 | ✅ 200；帶舊 `If-Match` → 409；不存在 → 404 |
| 結果 | ✅ 通過（額外加上樂觀鎖防 lost update） |

### ⑤ 刪除通知 — `DELETE /notifications/{id}`
| 項目 | 內容 |
|---|---|
| 需求 | 從 MySQL 刪除；移除 Redis 快取；成功 204；不存在 404 |
| 實作 | `NotificationService.delete`（刪 DB → evict by-id → recent 失效；不存在 404） |
| 狀態碼 | 204 / 404 |
| 測試 | `NotificationServiceTest.delete_evictsByIdAndRecent`、`delete_missing_throws404`、`NotificationControllerTest.delete_*` |
| 實機 | ✅ 204；刪除後 MySQL 無列、Redis `notification:{id}` 也移除；再查 404 |
| 結果 | ✅ 通過 |

## 七、如何重現驗證

```bash
docker compose up -d            # 起依賴（首次或改 schema 後加 down -v）
./mvnw test                     # 32 單元測試（免 infra）
./mvnw verify                   # + 整合測試（需 Docker）
./mvnw spring-boot:run          # 啟動服務
bash scripts/smoke-test.sh      # 11 項端到端檢查
```
